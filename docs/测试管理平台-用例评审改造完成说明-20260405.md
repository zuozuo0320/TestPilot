## 1. 改造结论

本轮“用例评审能力收敛”已完成，系统当前统一口径如下：

1. 所有评审动作统一收口到“用例评审模块（case-review）”执行。
2. 用例列表页不再承担直接评审职责，只保留评审结果展示、筛选、查看评审、加入评审入口。
3. `test_cases.review_result` 继续保留，但其值由用例评审模块统一回写，不再允许通过 testcase 创建/编辑接口直接维护。
4. testcase 维度 legacy 直评接口已从后端路由中彻底移除，不再提供兼容入口。

一句话总结：

- 用例列表：看结果、做管理、进评审
- 用例评审模块：建计划、执行评审、沉淀记录

## 2. 本轮完成项

### 2.1 前端完成项

- 移除 `TestFront/src/views/TestCasePage.vue` 中的直接提审 / 通过 / 驳回操作。
- 移除 `TestFront/src/api/testcase.ts` 中 legacy 直评 API 封装。
- 在用例列表页增加：
  - 单条“加入评审”
  - 单条“查看评审”
  - 批量“加入评审”
- 在用例编辑抽屉中，将评审信息改为只读展示，不再允许人工直接修改评审结果。
- 在 `TestFront/src/views/CaseReviewPage.vue` 中支持路由意图：
  - `create=1&testcaseIds=...`：打开新建评审并预选用例
  - `reviewId=...`：直接打开评审详情

### 2.2 后端完成项

- testcase 列表投影增加以下评审摘要字段：
  - `in_review`
  - `current_review_id`
  - `current_review_name`
  - `related_review_count`
- testcase 创建 / 编辑接口不再接收 `review_result` 作为业务输入。
- legacy 直评链路已彻底移除：
  - 路由移除
  - handler 移除
  - service 方法移除
- 用例评审结果仍由 `case-review` 模块统一回写 testcase 主表。

### 2.3 文档完成项

- 已将 legacy 接口在相关说明文档中统一标记为“已移除”。
- 已将用例评审 OpenAPI 文档同步为当前实际口径。
- 已补充本完成说明，作为当前阶段交付基线文档。

## 3. 已移除接口

以下接口已从后端路由中删除：

- `POST /projects/:projectID/testcase/:testcaseID/submit-review`
- `POST /projects/:projectID/testcase/:testcaseID/approve-review`
- `POST /projects/:projectID/testcase/:testcaseID/reject-review`

当前若继续调用上述接口，应返回 `404 Not Found`。

## 5. 当前数据口径

### 5.1 testcase 列表评审相关字段

- `review_result`：最终评审结果投影
- `in_review`：当前是否存在进行中的评审
- `current_review_id`：当前关联评审计划 ID
- `current_review_name`：当前关联评审计划名称
- `related_review_count`：历史关联评审计划数量

### 5.2 口径说明

- `review_result` 负责展示“最终结果”
- `in_review` / `current_review_*` 负责展示“当前上下文”
- 用例页面不再通过保存表单去改写评审结果

## 7. 对联调方的影响

### 7.1 前端 / 联调方

- 不要再调用 testcase 维度的 `submit-review / approve-review / reject-review`
- 如需发起评审，请统一走 case-review 计划接口
- 如需查看当前评审上下文，请使用 testcase 列表返回的评审摘要字段

### 7.2 测试方

- 需按新的业务路径回归：
  - 用例列表进入评审
  - 评审模块执行评审
  - 评审结果回写 testcase 列表
