# 测试管理平台-测试智编模块 OpenAPI 接口文档

> 版本：v0.1（创建于 2026-03-28）
> 关联文档：`测试管理平台-测试智编模块数据与接口设计-20260328.md`
> 文档定位：用于前后端联调、后端接口实现、Swagger/OpenAPI 导入前的接口契约定稿

## 1. 说明

本文档采用 `OpenAPI 3.0.3 / Swagger` 风格组织，目标是把当前接口草案进一步收敛成可联调的接口契约。

说明：
- 认证方式默认沿用平台既有登录态，文档中以 `bearerAuth` 占位表示
- 响应结构沿用统一包装：`code + message + data`
- 错误码沿用当前已确认的 `AI_SCRIPT_xxxx`
- 首期默认生成模式为 `RECORDING_ENHANCED`
- 导出脚本接口仅对具备导出权限的角色开放；已确认 `developer` 不允许导出脚本
- 首期脚本框架固定为 `Playwright`，脚本语言固定为 `TypeScript`，默认导出文件后缀为 `.spec.ts`

## 3. 接口补充说明

### 3.1 统一认证

建议统一复用平台既有登录态认证机制：
- 若平台已有 `Cookie + Session` 方案，Swagger 展示层可映射为相应安全方案
- 若平台已有 `JWT / Bearer Token` 方案，可直接采用本文档中的 `bearerAuth`

### 3.2 状态与权限联动

关键约束：
- `POST /ai-script/scripts/{scriptId}/confirm` 必须要求当前脚本版本 `validation_status = PASSED`
- `POST /ai-script/scripts/{scriptId}/validate` 在当前脚本已处于 `VALIDATING` 时必须拒绝重复触发
- `POST /ai-script/tasks/{taskId}/cases/update` 更新后仍必须至少绑定 1 条用例
- `POST /ai-script/scripts/{scriptId}/edit` 必须生成新版本，不覆盖原版本
- `POST /ai-script/tasks/{taskId}/execute` 仅允许 `generation_mode = AI_DIRECT`
- `POST /ai-script/tasks/{taskId}/recording/start` 与 `POST /ai-script/tasks/{taskId}/recording/finish` 仅允许 `generation_mode = RECORDING_ENHANCED`

### 3.3 前端按钮态建议

任务详情接口建议直接返回：
- `can_execute`
- `can_validate`
- `can_edit`
- `can_confirm`
- `can_export`
- `can_discard`

这样前端不需要重复拼装复杂的权限 + 状态判断逻辑。

### 3.4 导出脚本权限说明

已确认：
- `admin / manager / tester / reviewer` 可调用导出接口
- `developer / readonly` 不可调用导出接口
- 前端对无权限角色不展示导出按钮；后端对越权调用返回 `403`

## 4. 错误码补充

沿用当前已确认错误码：
- `AI_SCRIPT_4001` 参数校验失败
- `AI_SCRIPT_4002` 未关联测试用例
- `AI_SCRIPT_4003` 任务不存在
- `AI_SCRIPT_4004` 脚本版本不存在
- `AI_SCRIPT_4005` 当前状态不允许执行该操作
- `AI_SCRIPT_4006` 当前用户无权限
- `AI_SCRIPT_4007` 脚本尚未验证通过，不允许确认
- `AI_SCRIPT_4008` 当前版本不是任务主版本
- `AI_SCRIPT_4009` 验证中，不允许重复触发
- `AI_SCRIPT_4010` 已废弃版本不允许操作
- `AI_SCRIPT_5001` 生成任务执行失败
- `AI_SCRIPT_5002` 回放验证执行异常
