# 测试管理平台需求文档（当前基线）

> 版本：v2.4（更新于 2026-03-19）
> 项目：Aisight（后端 TestPilot / 前端 TestFront）
> 适用对象：产品、研发、测试、运维

## 1. 项目目标

构建一套工业级测试管理平台，覆盖：
- 测试用例管理（目录树、筛选、分页、编辑）
- 系统管理（用户/角色/项目）
- IAM 权限与审计闭环
- 个人中心与头像管理
- Docker 真环境稳定部署

## 2. 业务范围

### 2.1 In Scope（范围内）
1. 用户与角色管理
2. 项目管理与项目切换联动
3. 用例列表、筛选、分页、目录树管理
4. 用例创建/编辑（基础信息、状态、步骤）
5. 个人资料维护与头像上传
6. 基础审计日志（关键变更）

### 2.2 Out of Scope（暂不纳入）
1. 自动化执行引擎（仅预留 E2E 导航）
2. 第三方缺陷系统双向集成
3. 完整工作流编排引擎

## 3. 关键业务规则（已落地）

1. 预置系统角色：`admin / manager / tester / reviewer / readonly`
2. 预置角色不可删除（返回 409）
3. admin 用户不可删除（前后端双重限制）
4. 用户创建时：
   - 不可分配 admin 角色
   - 必须校验 name / email / phone
   - 头像不允许创建时设置（默认头像）
5. 个人中心：
   - 支持头像上传（jpg/jpeg/png/webp，<=2MB）
   - 邮箱不可修改（后端硬拒绝）
6. 用户-角色为多对多；用户-项目为多对多
7. 用户删除为逻辑删除（`deleted_at`）
8. 目录树规则：
   - 单根"全部用例"
   - 最多 5 层
   - 仅允许从叶子目录删除
   - 删除目录会级联删除目录下用例

## 5. 非功能需求（NFR）

| 编号 | 需求点 | 类别 |
|------|--------|------|
| NFR-01 | Docker 启动后 `/health` 返回 `ok` | 可用性 |
| NFR-02 | Seed 幂等，避免重复键导致服务崩溃 | 稳定性 |
| NFR-03 | 前后端统一业务校验（如 admin 禁配） | 一致性 |
| NFR-04 | 关键操作具备服务端约束，不依赖前端 | 安全性 |
| NFR-05 | 核心模块通过 `go test ./...` 与 `npm run build` | 可维护性 |

## 6. 验收标准（摘要）

1. 删除预置角色返回 409
2. 删除 admin 用户返回 409，前端按钮禁用
3. 新建用户不可选 admin，字段校验生效
4. 个人中心邮箱不可编辑，后端拒绝邮箱变更
5. 头像上传成功后可展示，失败有回退
6. 刷新后停留当前导航与项目，数据自动加载
7. 目录删除提示保留删除数量
