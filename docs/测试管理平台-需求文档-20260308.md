# 测试管理平台需求文档（当前基线）

> 版本：v2.2（更新于 2026-03-08）
> 项目：TestPilot / TestFront
> 适用对象：产品、研发、测试、运维

## 1. 项目目标

构建一套工业级测试管理平台，覆盖：
- 测试用例管理（目录树、筛选、分页、编辑）
- 系统管理（用户/角色/项目）
- IAM 权限与审计闭环
- 个人中心与头像管理
- Docker 真环境稳定部署

## 2. 业务范围

### 2.1 In Scope（范围内）
1. 用户与角色管理
2. 项目管理与项目切换联动
3. 用例列表、筛选、分页、目录树管理
4. 用例创建/编辑（基础信息、状态、步骤）
5. 个人资料维护与头像上传
6. 基础审计日志（关键变更）

### 2.2 Out of Scope（暂不纳入）
1. 自动化执行引擎（仅预留 E2E 导航）
2. 第三方缺陷系统双向集成
3. 完整工作流编排引擎

## 3. 关键业务规则（已落地）

1. 预置系统角色：`admin / manager / tester / reviewer / readonly`
2. 预置角色不可删除（返回 409）
3. admin 用户不可删除（前后端双重限制）
4. 用户创建时：
   - 不可分配 admin 角色
   - 必须校验 name / email / phone
   - 头像不允许创建时设置（默认头像）
5. 个人中心：
   - 支持头像上传（jpg/jpeg/png/webp，<=2MB）
   - 邮箱不可修改（后端硬拒绝）
6. 用户-角色为多对多；用户-项目为多对多
7. 用户删除为逻辑删除（`deleted_at`）
8. 目录树规则：
   - 单根“全部用例”
   - 最多 5 层
   - 仅允许从叶子目录删除
   - 删除目录会级联删除目录下用例

## 4. 功能需求（FR）

### FR-01 登录与会话
- 支持账号密码登录
- 登录后加载项目与首页数据
- 刷新页面需保持导航位置与当前项目

### FR-02 用户管理
- 支持用户列表分页、创建、编辑、删除（逻辑）
- 列表隐藏 ID 字段
- admin 删除按钮置灰禁用

### FR-03 角色管理
- 支持角色创建/编辑/删除
- 预置角色禁止删除

### FR-04 项目管理
- 支持项目列表与项目创建
- 顶部项目切换联动用例数据

### FR-05 用例列表
- 支持分页、关键字、等级、评审结果、执行结果筛选
- 支持排序与空状态展示

### FR-06 用例目录树
- 支持目录创建/删除
- 删除前确认；提示删除数量
- 目录与“全部用例”视觉统一

### FR-07 用例编辑
- 抽屉式编辑页面
- 分区：基础信息 / 评审与执行 / 步骤
- 步骤操作当前保留：复制、删除

### FR-08 个人中心
- 展示个人信息
- 支持头像上传
- 禁止邮箱修改

## 5. 非功能需求（NFR）

1. 可用性：Docker 启动后 `/health` 返回 `ok`
2. 稳定性：seed 幂等，避免重复键导致服务崩溃
3. 一致性：前后端统一业务校验（如 admin 禁配）
4. 安全性：关键操作具备服务端约束，不依赖前端
5. 可维护性：核心模块通过 `go test ./...` 与 `npm run build`

## 6. 验收标准（摘要）

1. 删除预置角色返回 409
2. 删除 admin 用户返回 409，前端按钮禁用
3. 新建用户不可选 admin，字段校验生效
4. 个人中心邮箱不可编辑，后端拒绝邮箱变更
5. 头像上传成功后可展示，失败有回退
6. 刷新后停留当前导航与项目，数据自动加载
7. 目录删除提示保留删除数量
