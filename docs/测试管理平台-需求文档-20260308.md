# 测试管理平台需求文档（当前基线）

> 版本：v2.5（更新于 2026-03-21）
> 项目：Aisight（后端 TestPilot / 前端 TestFront）
> 适用对象：产品、研发、测试、运维

## 1. 项目目标

构建一套工业级测试管理平台，覆盖：
- 测试用例管理（目录树、筛选、分页、编辑）
- 系统管理（用户/角色/项目）
- IAM 权限与审计闭环
- 个人中心与头像管理
- Docker 真环境稳定部署

## 2. 业务范围

### 2.1 In Scope（范围内）
1. 用户与角色管理
2. 项目管理与项目切换联动
3. 用例列表、筛选、分页、目录树管理
4. 用例创建/编辑（基础信息、状态、步骤）
5. 个人资料维护与头像上传
6. 基础审计日志（关键变更）

### 2.2 Out of Scope（暂不纳入）
1. 自动化执行引擎（仅预留 E2E 导航）
2. 第三方缺陷系统双向集成
3. 完整工作流编排引擎

## 3. 关键业务规则（已落地）

1. 预置全局系统角色：`admin / manager / tester / reviewer / developer / readonly`（详见 2.2 角色定义）
2. 预置角色不可删除（返回 409）
3. admin 用户不可删除（前后端双重限制）
4. 用户创建时：
   - 必须至少选择 1 个角色，支持一次选择多个角色
   - 角色选择为多选，权限按所选角色并集生效
   - 不可分配 admin 角色
   - 必须校验 name / email / phone
   - 头像不允许创建时设置（默认头像）
5. 个人中心：
   - 支持头像上传（jpg/jpeg/png/webp，<=2MB）
   - 邮箱不可修改（后端硬拒绝）
6. 用户-角色为多对多；用户-项目为多对多；角色为全局角色，与项目成员关系解耦
7. 用户角色的分配、调整、回收仅允许 `admin` 执行
8. 用户删除为逻辑删除（`deleted_at`）
9. 目录树规则：
   - 单根"全部用例"
   - 最多 5 层
   - 仅允许从叶子目录删除
   - 删除目录会级联删除目录下用例

### 3.2 跨模块联动规则（用户 ↔ 角色 ↔ 项目）

| 序号 | 触发操作 | 联动行为 |
|------|---------|---------|
| 1 | 用户被逻辑删除 | 自动从所有项目成员列表中移除；该用户创建的用例/缺陷的 `created_by` 展示为「用户名（已注销）」 |
| 2 | 用户被禁用 | 禁止后续登录，当前在线会话立即失效并强制退出；项目成员关系、数据归属均保持不变 |
| 3 | 项目归档 | 成员不再能在项目选择器中看到该项目（admin 除外），数据只读 |
| 4 | 删除自定义角色 | 需先解除所有用户关联；用户至少保留一个角色，不允许出现无角色用户 |
| 5 | 创建用户 | 默认绑定「AiSight Demo」项目，该项目不设成员上限 |
| 6 | 用户从项目中移除 | 其在该项目下创建的用例/缺陷归属不变，仅收回访问权限 |
| 7 | 用户角色调整（增/减/升/降） | 仅 `admin` 可操作；变更立即生效，已创建的数据保留，但按新角色权限控制后续操作 |
| 8 | 删除空项目 | 同步清理所有成员关系记录 |

### 模块一：登录与会话

| 编号 | 需求点 | 优先级 |
|------|--------|--------|
| FR-01-1 | 支持账号密码登录 | P0 |
| FR-01-2 | 登录后加载项目与首页数据 | P0 |
| FR-01-3 | 刷新页面保持导航位置与当前项目 | P0 |
| FR-01-4 | 非 `admin` 账号连续输错密码 5 次后锁定 1 小时，仅 `admin` 可手动解锁 | P0 |
| FR-01-5 | 登录后连续 20 分钟无任何有效操作自动退出；任意有效操作刷新会话空闲时间 | P0 |


#### 1.1 登录与会话业务规则

1. 非 `admin` 账号连续输错密码 5 次后锁定 1 小时
2. `admin` 账号永远不会因连续输错密码而被锁定
3. 成功登录后，登录失败计数立即清零
4. `admin` 手动解锁后，登录失败计数立即清零
5. 因连续登录失败被锁定的非 `admin` 用户仅允许由 `admin` 手动解锁
6. 登录成功后，如连续 20 分钟无任何有效操作，则自动退出登录
7. 有效操作包括：页面点击、输入、提交、路由切换、触发接口请求；仅停留页面不计为有效操作
8. 用户被禁用后，当前所有在线会话立即失效并强制退出

##### 2.2.1 预置角色定义

| 角色标识 | 显示名称 | 定位 | 典型用户 |
|---------|---------|------|----------|
| `admin` | 系统管理员 | 全权限，系统配置、用户/角色/项目管理 | 运维负责人、平台管理员 |
| `manager` | 项目管理员 | 项目级管理权限，可管理项目成员、配置模块 | 测试经理、QA Lead |
| `tester` | 测试工程师 | 用例增删改、执行、提交缺陷、导入导出 | 测试执行人员 |
| `reviewer` | 评审员 | 评审用例、修改评审状态，不可增删用例 | 开发 TL、产品经理 |
| `developer` | 开发工程师 | 查看用例+缺陷，认领/修复缺陷，不可改用例 | 研发人员 |
| `readonly` | 只读访客 | 纯查看，不可修改任何数据 | 外部审计、上级领导 |

##### 2.2.3 角色业务规则

1. 平台采用全局角色模型，角色不随项目切换而变化
2. 用户-角色为多对多关系，一个用户可同时拥有多个角色，取并集权限
3. 用户角色的分配、调整、回收仅允许 `admin` 执行；`manager` 仅能管理项目成员，不可修改角色
4. 用户创建时必须至少选择 1 个非 `admin` 角色，支持一次选择多个角色
5. 用户创建时角色选择为多选，最终权限按所选角色的并集生效
6. 用户角色的分配仅允许 `admin` 执行；用户创建时不可分配 `admin` 角色（仅 `admin` 可在编辑中授予）
7. 预置角色不可删除（后端返回 409）；自定义角色删除前需检查关联用户，有关联则禁止删除并提示
8. 自定义角色不允许配置用户管理、角色管理、项目创建、项目删除等系统级权限
9. 角色权限被修改后，所有关联用户立即生效，无需重新登录，并记录审计日志
10. 用户仅由管理员创建，不开放自主注册
11. `developer` 角色的核心定位是「缺陷协作」，不可修改用例，但可查看所有测试数据
12. 数据可见性：所有角色均可看到当前项目内的全部用例，不按创建人隔离
13. 邮箱在任何场景下均不可修改（用户管理编辑、个人中心，后端统一硬拒绝）
14. 用户必须绑定至少一个项目（`admin` 角色除外，admin 自动拥有所有项目访问权）
15. 个人中心所有用户均可访问，可查看自己的角色、关联项目等信息
16. 无权限访问页面时，前端不跳转，直接在页面内展示占位符提示「无权限访问该页面，请联系管理员」

##### 2.3.1 项目业务规则

1. 项目名称全局唯一，创建/编辑时前后端双重校验
2. 归档项目内所有数据变为只读（用例、目录、缺陷、执行记录均不可增删改）
3. 仅 `admin` 可恢复已归档项目为活跃状态
4. 仅 `admin` 可删除项目，且必须满足：已归档 + 无用例 + 无缺陷
5. 新建项目后自动创建根目录「全部用例」
6. 项目选择器按权限过滤：普通用户只看到已加入的项目，admin 看到全部
7. 项目成员关系仅用于控制项目访问范围，不承载角色语义；项目成员管理不改变用户全局角色
8. 项目中带 `admin` / `manager` 全局角色的成员为受保护成员，不可被移除；`admin` 和 `manager` 仅可移除其他成员
9. 切换项目后前端自动刷新所有业务模块数据，不保留上一个项目的缓存状态
10. 种子项目「AiSight Demo」不允许删除和归档（前后端双重限制）
11. 仪表盘数据范围：`admin` / `manager` 看所有项目汇总，其他角色只看当前选中项目的统计
12. 项目列表默认按最后活跃时间降序，归档项目始终排在底部

##### 3.2.1 目录与列表联动规则

1. 目录用于对相关用例进行分组和归类
2. 点击任一目录节点时，用例列表默认展示该目录及其所有子目录下的用例
3. 目录删除仅允许删除空叶子目录；非空目录不可删除
4. 用例未绑定目录时，统一展示在“未规划用例”下
5. “未规划用例”为系统内置目录，不允许重命名、删除、拖拽移动
6. 普通目录重命名后，需同步影响其所有子目录路径及该目录下所有已归属用例的目录归属
7. 普通目录仅允许同级拖拽排序，不允许拖拽移动到其他父目录下
8. 目录后显示的数量为当前目录直属用例数，不包含子目录下的用例数量
9. “全部用例”节点后显示的数量为项目下全部用例总数
10. 目录搜索仅用于目录树节点定位，不直接改变当前用例列表结果
11. 搜索命中目录时，应展示匹配节点并自动展开其父级路径，便于快速定位
12. 用户点击搜索结果中的目录节点后，用例列表再切换为该目录及其所有子目录下的用例
13. 搜索无结果时，提示“未找到匹配目录”，且当前用例列表保持不变
14. 切换目录时，保留当前关键字、状态、评审结果、执行结果、等级、标签、创建人、更新人、时间范围、排序方式等筛选条件，仅替换目录条件
15. 切换目录后，列表页码重置为第 1 页，当前勾选状态清空
16. 点击“全部用例”时，仅取消目录限制，其他筛选条件保持不变
17. 若当前筛选条件导致目标目录下无数据，则展示空结果，不自动清空筛选条件

##### 3.3.1 用例生命周期与版本规则

1. 用例状态定义为：`草稿 / 待评审 / 已生效 / 已废弃`
2. 评审结果定义为：`未评审 / 已通过 / 未通过`；执行结果定义为：`未执行 / 成功 / 失败`
3. 新建用例默认值为：状态=`草稿`，评审结果=`未评审`，执行结果=`未执行`，版本号=`V1`
4. `tester` / `manager` / `admin` 可将草稿用例提交评审（`草稿 -> 待评审`）
5. `reviewer` / `manager` / `admin` 可执行评审；评审通过时 `待评审 -> 已生效` 且评审结果置为 `已通过`；评审不通过时 `待评审 -> 草稿` 且评审结果置为 `未通过`
6. 待评审用例可由 `tester` / `manager` / `admin` 撤回到草稿
7. 草稿用例仅允许 `tester` / `manager` / `admin` 删除；待评审用例需先撤回草稿再删除；已生效用例不可删除，只能废弃
8. 已生效用例仅允许 `manager` / `admin` 废弃为 `已废弃`；已废弃用例默认隐藏，仅可通过状态筛选或「包含已废弃」开关显示
9. 已废弃用例为只读，不可编辑、提审、执行，也不可直接删除；仅 `manager` / `admin` 可恢复为草稿
10. 已废弃用例恢复为草稿后，评审结果重置为 `未评审`，最新执行结果重置为 `未执行`，历史评审记录与执行记录保留
11. `developer` / `readonly` 在用例全生命周期内均为只读，不可编辑、删除、提审、评审、废弃、恢复或执行
12. 实质变更字段包括：用例名称、用例等级、前置条件、步骤、预期结果、备注、附件；目录移动、标签调整、排序变化不属于实质变更
13. 用例发生实质变更并保存后，版本号自动 `+1`；若当前状态为 `已生效`，则自动回到 `草稿`，评审结果重置为 `未评审`，最新执行结果重置为 `未执行`
14. 每次评审都必须填写评审意见，并记录评审人、评审时间、评审结果、评审意见、对应用例版本号
15. 每次执行都必须保留历史记录，并记录执行人、执行时间、执行结果、执行备注、对应用例版本号；仅 `tester` / `manager` / `admin` 可写执行记录
16. 用例列表默认展示：用例名称、状态、版本号、用例等级、所属模块、评审结果、执行结果、标签、更新人、更新时间；执行结果展示最新一次执行结果
17. 用例详情展示基础信息、当前状态、当前版本号、评审记录、执行记录、编辑历史
18. 用例导入仅允许新增，不允许通过导入覆盖或更新已有用例；导入允许部分成功，并返回总数、成功数、失败数及失败明细（至少包含行号、字段、失败原因）
19. 用例导入模板固定字段为：所属目录、用例名称、用例等级、前置条件、标签、备注，以及按列展开的步骤/预期结果对（步骤1/预期结果1、步骤2/预期结果2 ...）
20. 导入时至少需要 1 组有效步骤与预期结果；所属目录为空时默认导入到当前选中的目录
21. 导入不允许写入状态、评审结果、执行结果、版本号、评审记录、执行记录；附件不通过 Excel 导入，富文本内容首版按纯文本处理
22. 同一项目、同一目录下名称重复的用例允许导入，但结果中必须返回重复提醒，且不计入失败
23. 用例导出仅允许 `tester` / `manager` / `admin` 使用，且仅支持 xlsx
24. 用例导出严格遵循当前筛选条件，仅导出当前筛选结果
25. 导出默认字段为：用例名称、状态、版本号、用例等级、所属模块、评审结果、执行结果、标签、更新人、更新时间；执行结果为最新一次执行结果
26. 导出不包含评审记录、执行记录、编辑历史等明细数据，且导出的是当前最新版本数据，不包含历史版本快照
27. 已废弃用例默认不导出，仅在勾选「包含已废弃」后导出；导出文件名默认包含项目名和导出时间
28. 新建用例时，若当前选中的是具体目录节点，则所属目录默认带入该目录；若当前选中的是“全部用例”，则默认带入“未规划用例”
29. 编辑用例修改所属目录并保存后，目录树数量与当前用例列表应按最新目录和筛选条件立即刷新
30. 编辑用例修改所属目录后，该用例是否继续显示在当前列表中，取决于修改后的目录是否仍属于当前选中目录范围；若不再属于，则应从当前列表中消失
31. 批量移动用例时，允许选择“未规划用例”作为目标目录
32. 批量移动完成后，目录树数量与当前用例列表应按最新目录和筛选条件立即刷新
33. 批量移动后，若某用例移动后的目录不再属于当前选中目录范围，则该用例应立即从当前列表中消失；若仍属于，则继续显示

## 6. 验收标准（摘要）

1. 删除预置角色返回 409
2. 删除 admin 用户返回 409，前端按钮禁用
3. 新建用户必须至少选择 1 个非 `admin` 角色，支持多选，且字段校验生效
4. 用户编辑允许修改角色和关联项目，但不允许保存为无角色用户；非 `admin` 用户不允许保存为无项目用户
5. 自定义角色不可配置用户管理、角色管理、项目创建、项目删除等系统级权限
6. 角色权限变更后，关联用户立即生效，无需重新登录，且记录审计日志
7. 项目中带 `admin` / `manager` 角色的成员不可被移除
8. 用户被禁用后，当前在线会话立即失效并强制退出
9. 非 `admin` 账号连续输错密码 5 次后锁定 1 小时，仅 `admin` 可手动解锁；`admin` 账号不会被锁定
10. 成功登录或 `admin` 手动解锁后，登录失败计数清零
11. 登录后连续 20 分钟无有效操作自动退出；页面点击、输入、提交、路由切换、接口请求会刷新会话
12. 个人中心邮箱不可编辑，后端拒绝邮箱变更
13. 头像上传成功后可展示，失败有回退
14. 刷新后停留当前导航与项目，数据自动加载
15. 目录删除仅允许删除空叶子目录；非空目录不可删除
16. 点击父目录时，可查看当前目录及所有子目录下的用例

17. 新建用例默认状态为 `草稿`，评审结果为 `未评审`，执行结果为 `未执行`，版本号为 `V1`
18. 用例状态按角色规则流转：`tester` / `manager` / `admin` 可提审；`reviewer` / `manager` / `admin` 可评审；评审不通过回到草稿且必须填写评审意见
19. 已生效用例发生实质变更后，版本号自动 `+1`，并回到 `草稿`，评审结果重置为 `未评审`，最新执行结果重置为 `未执行`
20. 仅草稿用例可删除；已生效用例不可删除只能废弃；已废弃用例默认隐藏且可按状态筛选或「包含已废弃」开关显示
21. 已废弃用例仅 `manager` / `admin` 可恢复为草稿；恢复后历史评审记录与执行记录保留
22. 每次评审与每次执行都保留带版本号的历史记录；执行记录仅允许 `tester` / `manager` / `admin` 写入
23. 用例列表默认展示用例名称、状态、版本号、用例等级、所属模块、评审结果、执行结果、标签、更新人、更新时间，其中执行结果为最新一次执行结果
24. 用例详情展示基础信息、当前状态、当前版本号、评审记录、执行记录、编辑历史
25. 用例导入仅允许新增且允许部分成功；导入结果返回总数、成功数、失败数以及到行号/字段/原因级别的失败明细
26. 导入模板字段固定为所属目录、用例名称、用例等级、前置条件、标签、备注及按列展开的步骤/预期结果对；至少需要 1 组有效步骤与预期结果
27. 导入时所属目录为空默认进入当前选中目录；状态、评审结果、执行结果、版本号、评审记录、执行记录、附件均不可通过 Excel 导入
28. 同一项目同一目录下名称重复的用例可导入，但结果中需返回重复提醒，且不计入失败
29. 用例导出仅允许 `tester` / `manager` / `admin` 使用，且仅支持 xlsx
30. 导出严格遵循当前筛选条件，仅导出当前筛选结果；默认导出字段为用例名称、状态、版本号、用例等级、所属模块、评审结果、执行结果、标签、更新人、更新时间
31. 导出中的执行结果为最新一次执行结果，不包含评审记录、执行记录、编辑历史等明细数据，也不包含历史版本快照
32. 已废弃用例默认不导出，仅勾选「包含已废弃」时导出；导出文件名默认包含项目名和导出时间

33. 未绑定目录的用例统一展示在“未规划用例”下
34. “未规划用例”为系统内置目录，不允许重命名、删除、拖拽移动
35. 普通目录重命名后，其所有子目录和已归属用例的目录归属需同步更新
36. 普通目录仅允许同级拖拽排序，不允许拖拽移动到其他父目录下
37. 目录后显示的数量为当前目录直属用例数，不包含子目录下的用例数量
38. “全部用例”节点后显示的数量为项目下全部用例总数
39. 目录搜索仅定位目录节点，不直接改变当前用例列表；点击搜索结果目录后，列表再按该目录及其子目录联动
40. 目录搜索命中时自动展开命中节点及父级路径；无结果时提示“未找到匹配目录”且不清空当前列表
41. 切换目录时保留现有列表筛选条件，仅替换目录条件；点击“全部用例”时仅取消目录限制
42. 切换目录后列表页码重置为第 1 页且清空当前勾选状态；若目标目录无匹配数据则显示空结果且不自动清空筛选条件
43. 新建用例时，若当前选中的是具体目录节点，则所属目录默认带入该目录；若当前选中的是“全部用例”，则默认带入“未规划用例”
44. 编辑用例修改所属目录并保存后，目录树数量与当前用例列表应按最新目录和筛选条件立即刷新
45. 编辑用例修改所属目录后，若修改后的目录不再属于当前选中目录范围，则该用例应从当前列表中消失；若仍属于，则继续显示
46. 批量移动用例时，允许选择“未规划用例”作为目标目录
47. 批量移动完成后，目录树数量与当前用例列表应按最新目录和筛选条件立即刷新
48. 批量移动后，若某用例移动后的目录不再属于当前选中目录范围，则该用例应立即从当前列表中消失；若仍属于，则继续显示
