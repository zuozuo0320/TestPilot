## 4. 部分实现或与需求不一致

### 4.1 ~~目录树当前不是正式落库实现~~ [已解决]

> **2026-04-20 更新**：前端已切换为正式 `modules` 表落库方案，通过 `getModuleTree`、`createModule`、`renameModule`、`deleteModule` API 与后端交互. 目录持久化到数据库，多人共享、刷新可见均已实现.

### 4.2 ~~目录创建、重命名、删除与需求规则不一致~~ [已部分解决]

> **2026-04-20 更新**：前端已切换为正式 API 驱动的目录管理. 目录重命名、删除均通过后端 API 执行.
> 仍需关注的点：
> - 后端 `module` 删除接口是否已校验"目录下是否有用例"（仅空叶子目录可删）.
> - 后端 `module` 重命名是否同步更新子目录路径和已归属用例的 `module_path`.

### 4.3 目录统计口径与需求不一致

- 侧边栏普通目录后的数量，仍需确认是否已改为"直属用例数".
- "全部用例"节点是否已显示总数需确认.
- "未规划用例"节点数量展示口径是否已切换为后端统一计算需确认.

### 4.4 目录树功能部分仍未达到完整需求

- 未实现目录搜索.
- 未实现搜索命中后自动展开父级路径.
- 未实现"未找到匹配目录"提示.
- 未实现拖拽排序.
- 未实现同级排序持久化.
- 后端已存在 `moveModule` 路由（`PUT /projects/:projectID/modules/:moduleID/move`），但前端尚未接入拖拽排序交互.

### 4.5 新建用例默认目录与需求不一致

- 需求要求：若当前选中具体目录，新建用例默认带入当前目录；若选中"全部用例"，默认带入"未规划用例".
- 需确认当前 `openCreate()` 是否已读取当前选中目录.

### 4.6 列表字段与需求定义不一致

- 当前列表未实现“状态”字段.
- 当前列表未实现“版本号”字段.
- 当前列表展示了 `创建人 / 创建时间 / 操作`，但需求默认列要求的是“用例名称、状态、版本号、等级、所属模块、评审结果、执行结果、标签、更新人、更新时间”.
- 当前表头中的“状态”实际对应的是执行结果，不是用例生命周期状态.
- 当前“执行结果”只是直接展示 `exec_result` 字段，不是“执行记录中的最新一次执行结果”.

### 4.7 编辑页右侧信息部分已实现，部分仍为占位 [2026-04-21 更新]

- ~~"版本追踪"区块当前是静态占位数据~~ [已解决：已绑定真实数据]
- ~~"最近动态"区块当前是静态占位数据~~ [已解决：已接入 activities API 动态渲染]
- ~~"后置条件"当前只是占位~~ [已解决：已新增模型字段并前后端打通]
- "历史缺陷关联"当前仍是静态占位展示.
- "AI 智检助手"区块当前仍是静态占位，未接入实际 AI 服务（见 5.4）.

### 4.8 步骤编辑只做了部分

- 已实现新增、删除、拖拽排序.
- 未实现步骤复制.
- 未实现“在上方插入”.
- 未实现“在下方插入".
- 未实现步骤类型切换.

### 4.9 导入导出仅为基础骨架，和需求细则差距较大

- 导出接口当前没有接收列表筛选参数，实际是全量导出，不满足“严格遵循当前筛选条件”.
- 导出文件名当前是 `testcases_项目ID_时间.xlsx`，不包含项目名称.
- 前端页面上的“导出报告”按钮当前没有绑定测试用例导出逻辑.
- 页面里虽然有隐藏的导入 input，但没有清晰的可见导入入口.
- 当前导出字段包含 `ID / 标题 / 等级 / 模块 / 评审结果 / 执行结果 / 标签 / 前置条件 / 步骤 / 备注 / priority`，与需求中的默认导出字段不一致.
- 当前导入会读取评审结果、执行结果等字段，这与“导入不允许写入状态/评审结果/执行结果/版本号/记录数据”的需求不一致.
- 当前导入结果只返回 `created / skipped`，没有返回总数、失败数、失败明细、重复提醒.
- 当前导入没有“目录为空时导入到当前选中目录”的能力，后端默认会落到 `/未分类`.
- 当前导入模板不是“步骤1/预期结果1、步骤2/预期结果2...”成对列展开模型.

### 4.10 “未规划用例”与后端默认值不一致

- 需求确认后的目录名称是“未规划用例”.
- 当前前端逻辑使用 `/未规划用例`.
- 当前后端新建/更新/导入默认目录仍是 `/未分类`.
- 这会导致前后端对“未绑定目录用例”的归属定义不一致.

### 4.11 权限边界未按需求落细

- 当前测试用例、目录、附件、xlsx 接口基本都只校验“是否有项目访问权限”.
- 当前没有按 `tester / reviewer / manager / admin / developer / readonly` 做测试用例模块的细粒度权限控制.
- 当前导入导出没有按“仅 `tester` / `manager` / `admin` 可用”做后端限制.
- 当前 `developer / readonly` 也没有在后端被限制为只读.

### 4.12 ~~生命周期相关字段和规则没有真正建立~~ [已解决]

> **2026-04-20 更新**：
> - `TestCase` 模型已包含 `Status` 字段（`draft/pending/active/discarded`）和 `Version` 字段（默认 `V1`）.

> - 待评审用例发生实质变更时返回错误提示"请先撤回".
> - 已废弃用例编辑时返回错误提示"只读".
> - 评审结果回写由 `case-review` 模块统一执行，testcase 维度 legacy 直评接口已移除.

### 4.15 ~~更新人字段当前不会在编辑时刷新~~ [已解决]

### 4.16 名称唯一性规则比需求更严格

- 按现状，项目内任意目录的重名用例都会被数据库直接拦截.

   - TestPilot 后端代理转发请求到 Executor.

   - Executor: 新增接口、LLM prompt 模板.
