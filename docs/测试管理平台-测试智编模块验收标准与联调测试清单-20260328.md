# 测试管理平台-测试智编模块验收标准与联调测试清单

> 版本：v0.1（创建于 2026-03-28）
> 关联文档：`测试管理平台-测试智编模块需求方案-20260327.md`、`测试管理平台-测试智编模块详细设计-20260328.md`、`测试管理平台-测试智编模块数据与接口设计-20260328.md`、`测试管理平台-测试智编模块OpenAPI接口文档-20260328.md`
> 文档定位：用于功能提测、前后端联调、执行服务联调、上线前验收

## 1. 目的与适用范围

本文档用于统一“测试智编”模块首期开发完成后的验收口径，明确：
- 什么功能算开发完成
- 什么状态流转算符合预期
- 哪些接口必须联调通过
- 哪些异常场景必须覆盖
- 上线前最少应通过哪些检查项

适用范围：
- TestFront 前端
- TestPilot 平台后端
- 测试智编执行服务
- 与 MySQL、Redis/队列、对象存储的基础集成

不包含：
- Cypress 输出
- 复杂认证场景
- 完整在线编辑器
- 自动修复能力

## 2. 验收原则

### 2.1 总体验收原则

必须满足以下原则，才能判定首期功能可交付：
- 业务闭环完整：从创建任务到脚本确认全链路可走通
- 状态流转正确：任务状态、脚本状态、验证状态与规则一致
- 权限边界明确：不同角色看到的按钮、可执行操作符合预期
- 结果可追溯：轨迹、截图、日志、错误原因、验证结果均可查看
- 失败可恢复：生成失败、验证失败、验证异常后用户能继续推进
- 数据可落库：任务、脚本版本、验证记录、轨迹、证据、审计日志均可正确保存

### 2.2 通过标准

建议作为首期最低通过标准：
- P0 功能验收项全部通过
- P0 联调清单全部通过
- 不存在阻塞主流程的 P1/P2 缺陷
- 不存在状态错乱、权限越权、数据丢失、验证通过后无法确认等致命问题

## 3. 验收前准备

### 3.1 环境准备

至少准备以下环境与依赖：
- 平台前端环境 1 套
- 平台后端环境 1 套
- 测试智编执行服务环境 1 套
- MySQL 可用
- Redis / 队列可用
- 对象存储或文件存储可用
- `browser-use` 依赖环境可用
- `Playwright` 浏览器运行环境可用
- `Node.js` 运行环境可用，用于执行 `Playwright TypeScript` 脚本
- LLM 配置可用

### 3.2 测试数据准备

至少准备以下测试数据：
- 1 个可访问项目
- 至少 3 条测试用例
- 至少 1 个可正常执行的基础场景页面
- 1 个会在生成阶段失败的场景
- 1 个会在验证阶段失败的脚本样本
- `admin`、`manager`、`tester`、`reviewer`、`readonly` 各 1 个账号

### 3.3 验收角色建议

建议以下角色共同参与：
- 产品：确认业务行为和页面结果
- 前端：确认展示、按钮态、状态刷新
- 后端：确认接口、状态流转、数据落库
- 执行服务研发：确认生成与验证链路
- 测试：执行验收清单并记录问题

## 4. P0 功能验收标准

### 4.1 任务创建验收

验收项：
- 能进入 `测试智编` 模块并看到任务入口
- 能选择项目
- 能选择至少 1 条测试用例，且支持多选
- 创建任务时生成模式默认是 `录制增强模式`
- 可切换为 `AI直生模式`
- 能填写任务名称、场景描述、起始地址
- 表单缺失必填项时，无法提交并有明确提示
- 提交成功后任务进入 `待执行`

通过标准：
- 创建成功时返回任务 ID，页面列表可见新任务
- 提交后数据库存在任务记录与任务-用例关联记录

### 4.2 任务列表验收

验收项：
- 能查看任务列表
- 能按项目、任务状态、验证状态、生成人、关键字筛选
- 列表字段展示正确：任务名称、项目、关联用例数、框架、状态、版本号、验证状态、创建人、更新时间
- `复制配置`、`查看详情` 可用

通过标准：
- 列表数据与数据库一致
- 筛选条件生效，无脏数据或错乱状态

### 4.3 生成执行验收

验收项：
- 录制增强模式下可点击 `开始录制` 并进入 `执行中`
- AI直生模式下可点击 `执行生成` 并进入 `执行中`
- 执行成功后生成脚本草稿
- 生成失败时任务进入 `生成失败`
- 失败后能查看错误原因、最后截图、已采集轨迹
- 失败后可重新生成或复制配置新建任务

通过标准：
- 成功场景下任务状态更新正确，脚本版本成功入库
- 失败场景下任务状态更新为 `生成失败`，且错误信息可追溯

### 4.3A 录制增强验收

验收项：
- 能启动 `Playwright` 录制
- 能保存原始录制稿
- 能从录制稿中提取结构化步骤模型
- AI 能基于录制结果生成标准化脚本草稿
- 原始录制稿与最终脚本可关联查看

通过标准：
- 数据库存在录制会话记录
- 原始录制稿、步骤模型、最终脚本三者可追溯关联

### 4.4 轨迹与证据验收

验收项：
- 任务详情可查看轨迹时间线
- 轨迹至少展示动作类型、页面 URL、目标元素摘要、定位器、执行结果、时间戳
- 关键步骤有截图或页面证据
- 失败步骤有错误信息

通过标准：
- 轨迹条数与执行结果基本一致
- 截图/日志/证据可正常打开或下载

### 4.5 脚本生成验收

验收项：
- 首期仅生成 `Playwright TypeScript` 脚本
- 脚本内容可预览、复制、导出
- 脚本版本号正确生成
- 生成来源、生成时间、版本号展示正确
- 录制增强模式下能查看原始录制稿和标准化脚本差异
- 至少存在 1 个最终结果断言
- 能看到断言建议/待确认项

通过标准：
- 脚本版本正确落库
- 导出内容与页面显示内容一致
- 导出文件后缀默认符合 `.spec.ts` 约定
- 没有出现“只有动作没有校验”的脚本

### 4.6 回放验证验收

验收项：
- 只有用户手动点击 `验证` 才会触发回放验证
- 触发后验证状态进入 `验证中`
- 验证通过时状态变为 `验证通过`
- 验证失败时状态变为 `验证失败`
- 验证异常时状态变为 `验证异常`
- 能看到失败步骤、报错、截图、日志、断言结果

通过标准：
- 同一脚本处于 `验证中` 时不能重复触发验证
- 验证记录正确落库，并与脚本版本关联

### 4.7 脚本确认验收

验收项：
- `admin / manager / reviewer` 才能看到并执行 `确认脚本`
- `tester` 不可执行最终确认
- 验证未通过时脚本不可确认
- 验证通过后可确认
- 确认后脚本状态为 `已确认`
- 确认后任务状态为 `已确认`

通过标准：
- 未通过验证的确认请求被正确拒绝
- 通过验证的脚本确认后，状态、确认人、确认时间均正确写入

### 4.8 脚本有限编辑验收

验收项：
- 可对脚本草稿进行有限编辑
- 保存编辑后生成新版本，不覆盖原版本
- 编辑后新版本进入 `待确认`、`待重新验证`
- 已确认脚本再次编辑后，也必须生成新版本

通过标准：
- 历史版本仍可查看
- 新旧版本号正确递增

### 4.9 重新验证验收

验收项：
- 脚本重新生成后，原验证结果失效
- 人工编辑为新版本后，原验证结果失效
- 关键内容变更后，状态进入 `待重新验证`
- 非关键展示类变更不触发重验

通过标准：
- 命中重验条件时，脚本状态与任务状态同步进入 `待重新验证`
- 不命中重验条件时，验证状态保持不变

### 4.10 用例关联验收

验收项：
- 创建任务时必须关联至少 1 条用例
- 支持后续调整关联用例
- 用例详情可查看关联脚本列表
- 任务/脚本详情能正确展示关联用例信息

通过标准：
- 更新后仍不能出现 0 用例绑定
- 用例详情反查结果与任务、脚本数据一致

### 4.11 导出与审计验收

验收项：
- `admin / manager / tester / reviewer` 可导出脚本
- `developer / readonly` 不可导出脚本，前端不展示导出入口或调用后返回 `403`
- 创建任务、执行生成、验证、编辑、确认、废弃、导出均有审计记录

通过标准：
- 审计日志能定位到操作人、操作时间、操作类型
- 导出动作在日志中可追溯

## 5. 状态流转验收清单

### 5.1 任务状态

必须校验以下流转：
- `待执行 -> 执行中 -> 生成成功`
- `待执行 -> 执行中 -> 生成失败`
- `生成成功 -> 待确认`
- `待确认 -> 已确认`
- `待确认 -> 待重新验证`
- `待重新验证 -> 待确认`
- 任意可废弃状态 -> `已废弃`

### 5.2 脚本状态

必须校验以下流转：
- `草稿 -> 待确认`
- `待确认 -> 已确认`
- `已确认 -> 待确认 + 待重新验证（新版本）`
- `待确认 -> 待重新验证`
- 任意可废弃状态 -> `已废弃`

### 5.3 验证状态

必须校验以下流转：
- `未验证 -> 验证中 -> 验证通过`
- `未验证 -> 验证中 -> 验证失败`
- `未验证 -> 验证中 -> 验证异常`
- 命中重验条件 -> `未验证`

## 6. 权限验收清单

### 6.1 角色验收

至少校验以下角色差异：
- `admin`
- `manager`
- `tester`
- `reviewer`
- `readonly`

### 6.2 核心权限点

必须校验：
- `tester` 可创建、执行、编辑、触发验证，但不可最终确认
- `reviewer` 可查看并确认，但不可创建和执行生成
- `readonly` 仅可查看，不可创建、编辑、验证、确认
- 无权限用户调用接口时返回正确错误码或 403

说明：
- `developer` 不允许导出脚本，需作为正式上线验收项校验

## 7. 前后端联调测试清单

### 7.1 任务接口联调

必须联调通过：
- `POST /api/ai-script/tasks`
- `GET /api/ai-script/tasks`
- `GET /api/ai-script/tasks/{taskId}`
- `POST /api/ai-script/tasks/{taskId}/execute`
- `POST /api/ai-script/tasks/{taskId}/clone`
- `POST /api/ai-script/tasks/{taskId}/discard`

联调检查点：
- 请求参数与接口契约一致
- 返回字段与前端展示字段一致
- 错误码正确
- 状态变更后前端刷新正常

### 7.2 脚本接口联调

必须联调通过：
- `GET /api/ai-script/tasks/{taskId}/current-script`
- `GET /api/ai-script/tasks/{taskId}/versions`
- `POST /api/ai-script/tasks/{taskId}/edit-script`
- `POST /api/ai-script/tasks/{taskId}/recording/fail`
- `POST /api/ai-script/scripts/{scriptId}/confirm`
- `POST /api/ai-script/scripts/{scriptId}/discard`
- `GET /api/ai-script/scripts/{scriptId}/export`

联调检查点：
- 编辑后返回新版本信息
- 录制异常时能正确收口为失败状态，不会长期卡在 `RECORDING`
- 确认失败场景返回正确错误码
- 导出接口返回文件流或下载地址可用

### 7.3 验证接口联调

必须联调通过：
- `POST /api/ai-script/tasks/{taskId}/validate`
- `GET /api/ai-script/validations/latest?script_version_id={id}`
- `GET /api/ai-script/scripts/{scriptId}/validations`

联调检查点：
- 触发验证后立即进入 `VALIDATING`
- 完成后能刷新为最终验证状态
- `VALIDATING` 期间重复触发被拒绝
- 最近一次验证与验证历史数据一致

### 7.4 轨迹与证据接口联调

必须联调通过：
- `GET /api/ai-script/tasks/{taskId}/traces`
- `GET /api/ai-script/tasks/{taskId}/evidences`

联调检查点：
- 轨迹顺序正确
- 截图、日志、证据链接有效
- 失败场景下仍有可查看记录

### 7.5 用例关联接口联调

必须联调通过：
- `GET /api/cases/{caseId}/scripts`
- `POST /api/ai-script/tasks/{taskId}/cases/update`

联调检查点：
- 用例详情反查结果正确
- 更新后仍不能为 0 用例绑定
- 关键变更场景能触发重新验证逻辑

## 8. 执行服务联调清单

### 8.1 生成链路联调

必须验证：
- 平台后端可成功投递生成任务
- 执行服务可消费任务
- `browser-use` 执行完成后能回写轨迹、证据、脚本版本
- 失败场景下能回写失败原因

### 8.2 回放验证链路联调

必须验证：
- 平台后端可成功投递验证任务
- 执行服务可调用 `Playwright` 完成 `TypeScript` 脚本验证
- 验证结果能回写失败步骤、报错、截图、日志、断言结果
- 执行异常时状态正确写为 `验证异常`

### 8.3 存储联调

必须验证：
- 截图上传成功
- 日志上传成功
- 具备导出权限时，导出脚本文件可访问
- `developer / readonly` 调用导出接口返回 `403`
- 文件地址和元数据一致

## 9. 异常场景验收清单

必须覆盖以下异常：
- 创建任务缺少必填项
- 创建任务时未绑定用例
- 任务不存在
- 脚本版本不存在
- 当前状态不允许执行生成
- 当前状态不允许确认
- 验证中重复触发验证
- 已废弃脚本继续操作
- 回放验证异常
- 执行服务回调失败或超时
- 对象存储上传失败

通过标准：
- 页面有明确提示
- 接口有正确错误码
- 数据库不产生脏状态
- 审计日志可追溯

## 10. 数据落库验收清单

必须检查以下表：
- `ai_script_task`
- `ai_script_task_case_rel`
- `ai_script_version`
- `ai_script_validation`
- `ai_script_trace`
- `ai_script_evidence`
- `ai_script_operation_log`

检查点：
- 数据新增数量符合预期
- 状态字段与页面展示一致
- 版本号递增正确
- 审计日志完整
- 删除/废弃采用逻辑处理，不误删历史数据

## 11. 上线前通过标准

上线前至少应满足：
- P0 功能验收项全部通过
- 前后端联调清单全部通过
- 执行服务联调清单全部通过
- 数据落库检查全部通过
- 关键异常场景全部覆盖并通过
- 无阻塞主链路缺陷

## 12. 缺陷分级建议

建议按以下方式判定是否阻塞上线：
- P0：主流程无法走通、状态错乱、越权、数据丢失、验证通过仍无法确认
- P1：重要页面展示缺陷、非主链路功能异常、日志/证据缺失但主流程可走通
- P2：提示文案、展示细节、非关键交互体验问题
