## 1. 通用约定

### 1.1 基础路径

- 外部接口：`/api/v1`
- 内部接口（仅 executor 调用）：`/api/v1/internal`

### 1.2 鉴权

| 类型 | 鉴权方式 | 适用接口 |
|---|---|---|
| 用户接口 | `Authorization: Bearer {JWT}` | 全部 `/api/v1/...` 外部接口 |
| 内部回调 | `X-Internal-Token: {REQGEN_INTERNAL_TOKEN}` | 全部 `/api/v1/internal/...` 接口 |

JWT 中含 `user_id` 与 `role`，复用现有 `authMiddleware`。内部接口不走 `authMiddleware`，仅校验 token。

### 1.3 统一响应

成功（沿用平台规范）：

```json
{
  "code": 200000,
  "message": "success",
  "data": { ... },
  "request_id": "uuid"
}
```

分页：

```json
{
  "code": 200000,
  "data": [ ... ],
  "total": 123,
  "page": 1,
  "page_size": 20,
  "request_id": "uuid"
}
```

错误：

```json
{
  "code": 130001,
  "message": "需求文档不存在",
  "request_id": "uuid"
}
```

### 1.4 错误码段（13xxxx 需求智生）

| 码 | HTTP | 含义 |
|---|---|---|
| 130001 | 404 | 需求文档不存在 |
| 130002 | 400 | 需求文档格式不支持 |
| 130003 | 400 | 需求文档过大 |
| 130004 | 500 | 需求文档解析失败 |
| 130005 | 409 | 需求文档解析中，请稍候 |
| 130006 | 409 | 需求文档已删除 |
| 130101 | 404 | 生成任务不存在 |
| 130102 | 409 | 生成任务状态不允许此操作 |
| 130103 | 429 | 项目并发任务超限 |
| 130104 | 500 | LLM 调用失败（见 fail_reason 前缀） |
| 130105 | 412 | 激活模型未配置 |
| 130106 | 429 | 全局并发任务超限 |
| 130107 | 503 | Executor 不可用 |
| 130201 | 404 | Skill 不存在 |
| 130202 | 409 | Skill key 已存在 |
| 130203 | 403 | 系统 Skill 不可删除 |
| 130204 | 409 | Skill 已被他人修改，请刷新重试 |
| 130205 | 400 | Skill 提示词不符合规范 |
| 130301 | 409 | 产物已被采纳，不可重复采纳 |
| 130302 | 404 | 产物不存在 |
| 130303 | 409 | 产物已丢弃 |
| 130304 | 409 | 产物已变更，请刷新（乐观锁） |
| 130901 | 401 | 内部 token 鉴权失败 |
| 130902 | 409 | 回调任务已为终态，忽略 |

### 1.5 时间格式

ISO 8601，UTC：`2026-05-22T03:14:15Z`。响应中所有 datetime 字段都遵循此格式。

### 1.6 分页参数

| 参数 | 默认 | 说明 |
|---|---|---|
| `page` | 1 | 1-based |
| `page_size` | 20 | 最大 100 |

### 1.7 路径参数命名

沿用项目约定，全小写下划线：`:project_id`、`:doc_id`、`:task_id`、`:result_id`、`:skill_id`。

## 3. 需求文档接口

### 3.1 上传文件

**`POST /projects/:project_id/requirement-docs/upload`**

`Content-Type: multipart/form-data`

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `file` | file | ✓ | .docx/.pdf/.md/.txt，单文件 ≤ 10MB |
| `title` | string | ✗ | 默认取文件名去扩展名 |
| `remark` | string | ✗ | ≤ 500 字 |

**成功响应** `200`

```json
{
  "code": 200000,
  "data": {
    "id": 12,
    "project_id": 3,
    "title": "登录模块需求 v2",
    "source_type": "upload_file",
    "file_format": "docx",
    "file_size": 28471,
    "parse_status": "not_parsed",
    "created_by": 5,
    "created_at": "2026-05-22T03:14:15Z"
  }
}
```

**错误**

| 码 | 触发 |
|---|---|
| 130002 | 文件扩展名不在白名单 |
| 130003 | 文件 > 10MB |

**权限**：admin / manager / tester

**副作用**：写入文件到 `uploads/requirement_docs/{project_id}/{yyyy-mm}/{uuid}.{ext}`；INSERT `requirement_docs`；异步触发解析。

---

### 3.2 粘贴文本

**`POST /projects/:project_id/requirement-docs/paste`**

```json
{
  "title": "登录模块需求 v2",
  "content": "登录页要求...",
  "remark": ""
}
```

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `title` | string | ✓ | ≤ 200 字 |
| `content` | string | ✓ | ≤ 50 000 字 |
| `remark` | string | ✗ | ≤ 500 字 |

**成功响应** `200`：与 3.1 同结构，区别：

- `source_type=paste_text`
- `file_format=text`
- `file_path=""`
- `parse_status=parsed`（无需异步解析）

**权限**：admin / manager / tester

### 3.5 重新解析

**`POST /projects/:project_id/requirement-docs/:doc_id/reparse`**

无请求体。

**成功响应** `202`

```json
{
  "code": 200000,
  "data": {
    "id": 12,
    "parse_status": "parsing"
  }
}
```

**错误**：130005（当前正在解析中）/ 130001 / 130006。

**权限**：admin / manager / 上传人本人。

### 3.6 下载原文件

**`GET /projects/:project_id/requirement-docs/:doc_id/download`**

直接返回 `application/octet-stream` 二进制流，`Content-Disposition: attachment; filename="<title>.{ext}"`。

**错误**：130001 / 130006 / 粘贴文本类型返回 400 + message="该文档无原始文件"。

### 3.7 删除文档

**`DELETE /projects/:project_id/requirement-docs/:doc_id`**

**成功响应** `200`

```json
{ "code": 200000, "data": { "id": 12, "deleted_at": "2026-05-22T03:30:00Z" } }
```

**业务规则**：
- 软删（`deleted_at` 置时间）
- 已发起的 RUNNING 任务不中断
- 已落库的用例溯源链接显示为"原始需求已删除"
- 已 SUCCESS 的任务仍可查看产物与采纳

**权限**：admin / manager / 上传人本人。

**错误**：130001 / 130006。

## 4. 生成任务接口

### 4.1 创建任务

**`POST /projects/:project_id/requirement-gen/tasks`**

```json
{
  "task_name": "基于《登录模块需求 v2》生成",
  "requirement_doc_id": 12,
  "skill_id": 1,
  "target_module_id": 8,
  "default_level": "P2",
  "max_cases": 30,
  "extra_prompt": "侧重权限场景"
}
```

| 字段 | 类型 | 必填 | 默认 | 校验 |
|---|---|---|---|---|
| `task_name` | string | ✓ | — | ≤ 200 字 |
| `requirement_doc_id` | int | ✓ | — | 必须存在 + 未删除 + 同项目 |
| `skill_id` | int | ✓ | — | 必须 active + 项目可见（系统内置或本项目副本） |
| `target_module_id` | int | ✗ | 0 | 必须同项目 |
| `default_level` | string | ✗ | "P2" | P0/P1/P2/P3 |
| `max_cases` | int | ✗ | 30 | 1–50 |
| `extra_prompt` | string | ✗ | "" | ≤ 1000 字 |

**成功响应** `202`

```json
{
  "code": 200000,
  "data": {
    "id": 47,
    "task_name": "基于《登录模块需求 v2》生成",
    "status": "PENDING",
    "requirement_doc_id": 12,
    "skill_id": 1,
    "ai_model_config_id": 2,
    "ai_model_snapshot": "anthropic/claude-sonnet-4-6",
    "max_cases": 30,
    "created_at": "2026-05-22T03:20:00Z"
  }
}
```

**错误**

| 码 | 触发 |
|---|---|
| 130001 | requirement_doc_id 不存在或已删除 |
| 130103 | 项目并发任务 ≥ 3 |
| 130106 | 全局并发任务 ≥ 10 |
| 130105 | 无激活 AI 模型 |
| 130107 | Executor 健康检查失败 |
| 130201 | skill_id 不存在 |

**执行流**：
1. 校验入参 → 校验文档/Skill/模型
2. Redis Lua 扣减项目+全局配额
3. INSERT task (status=PENDING)
4. CAS 转 RUNNING
5. POST executor `/requirement/generate-cases`（含 callback_url）
6. 立即返回 task_id

### 4.5 批量采纳

**`POST /projects/:project_id/requirement-gen/tasks/:task_id/adopt`**

```json
{
  "result_ids": [301, 302, 305],
  "target_module_id": 8,
  "default_level_override": "P1"
}
```

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `result_ids` | int[] | ✓ | 1–50 个 |
| `target_module_id` | int | ✗ | 覆盖任务默认值 |
| `default_level_override` | string | ✗ | 覆盖单条 level（若条目原本未指定） |

**成功响应** `200`

```json
{
  "code": 200000,
  "data": {
    "adopted_count": 2,
    "adopted_case_ids": [201, 202],
    "skipped": [
      { "result_id": 305, "reason": "already_adopted", "case_id": 198 }
    ]
  }
}
```

**业务规则**：
- 每条采纳独立事务（防双写见 SQL 草案 §6.2）
- 已采纳 / 已丢弃的 result 跳过，不报错整批
- 同步累加 task.adopted_count；若 (adopted + discarded) ≥ generated → CAS 推进 task 状态

**错误**：130101 / 130102（任务非 SUCCESS / PARTIAL_ADOPTED 状态）。

## 6. 内部回调接口

仅 Executor 调用，鉴权方式：HTTP Header `X-Internal-Token: {REQGEN_INTERNAL_TOKEN}`。

### 6.1 任务心跳

**`POST /api/v1/internal/requirement-gen/tasks/:task_id/heartbeat`**

```json
{
  "executor_node_id": "executor-pod-3",
  "phase": "calling_llm",
  "progress": {
    "stage": "llm_streaming",
    "estimated_remaining_ms": 12000
  }
}
```

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `executor_node_id` | string | ✓ | executor 实例标识 |
| `phase` | string | ✓ | preparing / calling_llm / validating / persisting |
| `progress.stage` | string | ✗ | 任意子状态字符串 |
| `progress.estimated_remaining_ms` | int | ✗ | 仅用于前端进度估算 |

**业务规则**：
- 更新 `requirement_gen_tasks.last_heartbeat_at = NOW()`
- 同步写 Redis `reqgen:progress:{task_id}`（TTL 30s）供前端轮询命中
- 任务状态非 RUNNING 时返回 130902，executor 应停止后续工作

**成功响应** `200`

```json
{ "code": 200000, "data": { "accepted": true } }
```

**错误**：130101 / 130901 / 130902。

---

### 6.2 任务回调

**`POST /api/v1/internal/requirement-gen/tasks/:task_id/callback`**

**成功载荷**：

```json
{
  "executor_node_id": "executor-pod-3",
  "status": "SUCCESS",
  "duration_ms": 48200,
  "prompt_tokens": 1843,
  "completion_tokens": 5612,
  "model_used": "claude-sonnet-4-6",
  "result": {
    "cases": [ /* standard_case_v1 cases 数组 */ ],
    "summary": {
      "coverage_areas": [],
      "uncovered_risks": []
    }
  }
}
```

**失败载荷**：

```json
{
  "executor_node_id": "executor-pod-3",
  "status": "FAILED",
  "duration_ms": 12000,
  "prompt_tokens": 1843,
  "completion_tokens": 0,
  "fail_reason": "[BUSINESS] schema_violation: cases[2] missing field 'steps'"
}
```

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `executor_node_id` | string | ✓ | |
| `status` | string | ✓ | SUCCESS / FAILED |
| `duration_ms` | int | ✓ | |
| `prompt_tokens` | int | ✗ | 默认 0 |
| `completion_tokens` | int | ✗ | 默认 0 |
| `model_used` | string | ✗ | LLM 实际响应的模型名 |
| `result.cases` | array | status=SUCCESS 时必填 | 符合 standard_case_v1 |
| `result.summary` | object | ✗ | |
| `fail_reason` | string | status=FAILED 时必填 | 必须含 `[NETWORK]` / `[BUSINESS]` / `[SCHEMA]` / `[TIMEOUT]` 前缀 |

**幂等性**：

| 当前 status | 处理 |
|---|---|
| `RUNNING` | 接受，按 status 转 SUCCESS/FAILED；释放 Redis 配额 |
| `SUCCESS` / `FAILED` / `PARTIAL_ADOPTED` / `FULLY_CLOSED` | 返回 130902 + `{accepted: false}`，executor 停止重试 |
| `PENDING` | 不应发生；返回 130102 |

**成功响应** `200`

```json
{
  "code": 200000,
  "data": {
    "accepted": true,
    "task_status": "SUCCESS",
    "generated_count": 23
  }
}
```

**或忽略响应** `200`

```json
{
  "code": 130902,
  "message": "任务已为终态，回调忽略",
  "data": { "accepted": false, "current_status": "SUCCESS" }
}
```

**错误**：130101 / 130901 / 130102。

**业务规则**：
1. 鉴权（X-Internal-Token）
2. CAS 推进 status，避免重复回调写两次产物
3. status=SUCCESS：开事务批量 INSERT `requirement_gen_results`（每条独立 seq_no）；UPDATE task.status=SUCCESS + 计数字段
4. status=FAILED：UPDATE task.status=FAILED + fail_reason
5. 终态后释放 Redis 配额

## 10. 联调注意事项

### 10.1 顺序约束

- 上传文档 → 异步解析 → `parse_status=parsed` 后才能创建任务
- 任务 `status=SUCCESS` 后才能采纳/丢弃；`status=PENDING / RUNNING` 时仅能查看
- Skill 必须 `is_active=true` 且未软删才能被任务引用

### 10.2 幂等行为

| 操作 | 是否幂等 | 备注 |
|---|---|---|
| 上传文档 | 否 | 重复上传产生多条记录 |
| 重新解析 | 是 | CAS 状态转换，重复请求被吞 |
| 创建任务 | 否 | 重复请求产生多个任务 |
| 采纳产物 | 是 | 已采纳条目 skip |
| 丢弃产物 | 是 | 已丢弃条目 skip |
| 重跑任务 | 否 | 每次产生新任务 |
| 任务心跳 | 是 | 多次写入仅更新最新时间 |
| 任务回调 | 是 | CAS + 终态拒绝 |

### 10.3 前端轮询策略

- 文档解析：`parse_status` 为 `parsing` 时每 2s 轮询，最长 10 分钟
- 任务执行：`status` 为 `PENDING/RUNNING` 时每 2s 轮询，最长 10 分钟
- 用户主动停留页面才轮询；离开页面立即停止

### 10.4 大字段加载

- 文档列表不返回 `raw_content`（专用详情接口返回）
- Skill 列表不返回 `prompt_template`
- 产物列表始终返回完整字段（前端预览需要）
