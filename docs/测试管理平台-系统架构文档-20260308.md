# 测试管理平台系统架构文档（当前实现）

> 版本：v2.2（更新于 2026-03-08）
> 系统：TestPilot（后端） + TestFront（前端）

## 1. 总体架构

- 前端：Vue3 + TypeScript + Element Plus + Vite
- 后端：Go + Gin + GORM
- 数据库：MySQL
- 部署：Docker Compose

逻辑分层：
1. 表现层：Web UI（TestFront）
2. 接口层：REST API（Gin Router）
3. 领域服务层：用户/角色/项目/用例/目录逻辑
4. 数据访问层：GORM Models + MySQL
5. 运维层：Docker、seed、health

## 2. 关键模块

### 2.1 IAM 模块
- 用户管理：create/update/delete/list
- 角色管理：create/update/delete/list
- 用户角色绑定：多对多
- 用户项目绑定：多对多
- 关键约束：
  - 预置角色不可删除
  - admin 用户不可删除
  - 创建用户不可分配 admin

### 2.2 用例模块
- 用例列表：分页、筛选、排序
- 用例编辑：基础信息、状态、步骤
- 目录树：单根“全部用例”、最多 5 层、叶子优先删除
- 目录删除与用例级联清理

### 2.3 个人中心模块
- 个人资料更新（邮箱不可改）
- 头像上传：`POST /api/v1/users/me/avatar`
- 静态资源访问：`/uploads/*`

### 2.4 项目模块
- 项目列表、创建
- 顶部项目切换驱动用例查询
- 刷新后恢复当前项目

## 3. 核心接口（摘要）

1. 健康检查：`GET /health`
2. 项目：`GET /api/v1/projects`、`POST /api/v1/projects`
3. 用户：`GET/POST/PUT/DELETE /api/v1/users...`
4. 角色：`GET/POST/PUT/DELETE /api/v1/roles...`
5. 个人资料：`PUT /api/v1/users/me/profile`
6. 头像上传：`POST /api/v1/users/me/avatar`
7. 用例列表：`GET /api/v1/projects/{id}/testcases`（分页筛选）

## 4. 数据模型要点

- users（逻辑删除 `deleted_at`）
- roles（含预置系统角色）
- user_roles（多对多）
- projects
- user_projects（用户-项目绑定）
- project_members
- test_cases

## 5. 稳定性与部署设计

1. seed 幂等：对角色与默认数据采用可恢复策略，避免重复键崩溃
2. Docker 启动验证：容器 up + `/health=ok`
3. 404 旧镜像问题：通过 compose build 重建修复
4. 前端刷新状态恢复：导航、子菜单、项目 ID 持久化

## 6. 安全与风控

1. 服务端约束优先：关键删除与权限不可仅靠前端
2. 账号敏感字段保护：邮箱不可通过资料接口变更
3. 文件上传限制：类型/大小白名单
4. 审计能力：关键变更写入审计轨迹

## 7. 技术债与演进建议

1. 用例步骤拖拽与操作区进一步组件化
2. API 响应结构统一空值策略（items 空数组而非 null）
3. 增补头像上传接口自动化测试（成功/类型非法/超大小）
4. 推进 C4 文档化（Context/Container/Component）
