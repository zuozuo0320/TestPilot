# 测试管理平台-测试智编模块数据与接口设计（初稿）

> 版本：v0.1（创建于 2026-03-28）
> 关联文档：`测试管理平台-测试智编模块需求方案-20260327.md`、`测试管理平台-测试智编模块详细设计-20260328.md`
> 适用对象：后端研发、前端研发、测试、架构
> 文档定位：用于后端表结构设计、接口联调、状态枚举统一与研发任务拆分

## 1. 设计说明

本文档在已确认需求与详细设计基础上，继续细化：
- 数据表与字段草案
- 状态枚举与业务字典
- API 入参出参建议
- 关键错误码与校验规则
- 前后端联调口径

说明：
- 字段类型采用通用关系型数据库表达，可按最终技术栈映射为 MySQL / PostgreSQL 实际类型
- 所有时间字段统一建议使用 `datetime` 或等价时区安全类型
- 所有主键示例统一使用 `bigint` 或平台既有雪花 ID 方案
- 首期默认生成模式为 `RECORDING_ENHANCED`
- 首期唯一脚本输出框架固定为 `Playwright`，生成代码语言固定为 `TypeScript`
- 首期语言固定，不单独新增 `language_type` 字段；如后续支持多语言，再补充独立字段

## 4. 关键约束与联动规则

### 4.1 创建任务校验

必须满足：
- `project_id` 必填
- 至少关联 1 条测试用例
- `task_name` 必填
- `scenario_desc` 必填
- `start_url` 必填
- `generation_mode` 必填，默认值为 `RECORDING_ENHANCED`
- `framework_type` 首期固定为 `Playwright`
- 脚本正文默认产出 `TypeScript`，导出文件后缀默认 `.spec.ts`

### 4.2 生成成功后的落库顺序

建议顺序：
1. 写入或更新 `ai_script_task`
2. 若为录制增强模式，先写入 `ai_script_recording_session`
3. 写入 `ai_script_trace`
4. 写入 `ai_script_evidence`
5. 写入 `ai_script_version`
6. 回写任务当前脚本版本与状态
7. 写入操作日志

### 4.3 编辑脚本后的联动

保存为新版本时必须：
- `version_no + 1`
- 原版本保留，不覆盖
- 新版本 `script_status = PENDING_CONFIRM`
- 新版本 `validation_status = NOT_VALIDATED`
- 页面展示状态同步为 `待重新验证`
- 任务状态同步为 `PENDING_REVALIDATE`

### 4.4 确认脚本前校验

确认接口必须校验：
- 当前用户角色为 `admin / manager / reviewer`
- 当前脚本版本不为 `DISCARDED`
- 当前脚本版本 `validation_status = PASSED`
- 当前脚本版本为任务当前版本

### 4.5 重新验证触发后联动

命中重验场景时必须：
- 当前脚本版本 `validation_status` 置为 `NOT_VALIDATED`
- 当前脚本版本 `script_status` 置为 `PENDING_REVALIDATE`
- 任务状态置为 `PENDING_REVALIDATE`
- 原验证记录保留，不物理删除

## 5. API 统一约定

### 5.1 统一响应结构

建议统一响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

分页结构建议：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [],
    "page_no": 1,
    "page_size": 20,
    "total": 0
  }
}
```

### 5.2 通用错误码建议

| 错误码 | 说明 |
|--------|------|
| AI_SCRIPT_4001 | 参数校验失败 |
| AI_SCRIPT_4002 | 未关联测试用例 |
| AI_SCRIPT_4003 | 任务不存在 |
| AI_SCRIPT_4004 | 脚本版本不存在 |
| AI_SCRIPT_4005 | 当前状态不允许执行该操作 |
| AI_SCRIPT_4006 | 当前用户无权限 |
| AI_SCRIPT_4007 | 脚本尚未验证通过，不允许确认 |
| AI_SCRIPT_4008 | 当前版本不是任务主版本 |
| AI_SCRIPT_4009 | 验证中，不允许重复触发 |
| AI_SCRIPT_4010 | 已废弃版本不允许操作 |
| AI_SCRIPT_4011 | 仅允许删除已废弃任务 |
| AI_SCRIPT_5001 | 生成任务执行失败 |
| AI_SCRIPT_5002 | 回放验证执行异常 |

## 6. API 详细定义

### 6.1 POST /api/ai-script/tasks

说明：新建生成任务。

请求体：

```json
{
  "project_id": 1001,
  "task_name": "登录并进入用户列表",
  "generation_mode": "RECORDING_ENHANCED",
  "case_ids": [2001, 2002],
  "scenario_desc": "使用测试账号登录系统并进入用户列表页",
  "start_url": "https://test.example.com/login",
  "account_ref": "test_user_a",
  "env_config": {
    "browser": "chromium",
    "env_code": "test",
    "timeout_sec": 120
  },
  "remark": "首期验证登录基础流程"
}
```

响应体：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "task_id": 9001,
    "task_status": "PENDING_EXECUTE"
  }
}
```

说明：获取任务列表。

请求参数建议：
- `project_id`
- `task_status`
- `generation_mode`
- `validation_status`
- `created_by`

- `page_size`

## 7. 前后端联调口径

### 7.1 按钮显示与可点击分离

建议接口同时返回：
- `can_execute`
- `can_validate`
- `can_edit`
- `can_confirm`
- `can_export`
- `can_discard`
- `can_delete`

目的：
- 避免前端重复拼装复杂权限 + 状态判断
- 保持页面行为与后端真实规则一致

### 7.2 状态展示建议

前端展示建议至少保留两组状态：
- 任务状态
- 验证状态
- 生成模式

不要只用一个状态字段替代全部页面含义，否则容易混淆“待确认”和“待重新验证”。

### 7.3 长任务执行建议

生成任务执行与回放验证都属于长任务，建议：
- 提交接口快速返回受理结果
- 前端通过轮询详情接口或独立状态接口刷新
- 后续若平台已有消息推送机制，可再升级为事件推送
- 录制增强模式下，录制结束后的步骤模型提取与 AI 重构也按长任务处理
